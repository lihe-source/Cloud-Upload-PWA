## 🆕 v3.4 新功能

### 1. 三種更新策略

#### 🚀 自動更新（預設，推薦）

- 檢測到新版本後顯示倒數 10 秒
- 自動更新，無需手動點擊
- 可隨時取消

#### 🔔 顯示通知

- 顯示更新通知
- 由使用者決定何時更新
- 與 v3.3 相同

#### ✋ 手動更新

- 完全不顯示通知
- 由使用者自行重新載入
- 適合進階使用者

### 2. 更新設定介面

- Header 新增「更新設定」按鈕（🔄 圖標）
- 可隨時更改更新策略
- 設定自動儲存

### 3. 倒數計時視覺化

```
┌─────────────────────────────────────┐
│ 🚀  新版本 v3.4.1 已下載完成        │
│     將在 10 秒後自動更新...          │
│  [立即更新]  [取消自動更新]          │
└─────────────────────────────────────┘
```

## 📝 後續更新時需要修改的檔案

### ✅ 每次更新必須修改（只需 2 個檔案！）

#### 1️⃣ version.js

```javascript
const APP_VERSION = 'v3.4.0'; // ← 改成新版本號（例如 v3.4.1）
const BUILD_DATE = '2026-03-22'; // ← 改成更新日期
```

#### 2️⃣ service-worker.js

```javascript
const CACHE_VERSION = 'v3.4.0'; // ← 改成新版本號（例如 v3.4.1）
```

**⚠️ 重要：版本號必須一致！**

## 🚀 自動更新流程

### 使用者體驗（自動模式）

1. **使用者正常使用應用程式**
   
2. **開發者推送新版本到 GitHub**
   - 修改 version.js → v3.4.1
   - 修改 service-worker.js → v3.4.1
   - git push

3. **30 秒後，使用者看到**
   ```
   🚀 新版本 v3.4.1 已下載完成
   將在 10 秒後自動更新...
   ```

4. **10 秒倒數**
   ```
   將在 9 秒後自動更新...
   將在 8 秒後自動更新...
   ...
   將在 1 秒後自動更新...
   ```

5. **自動重新載入**
   - 頁面自動刷新
   - 套用新版本
   - 完成！✅

### 使用者可以

- ✅ 點擊「立即更新」→ 馬上更新
- ✅ 點擊「取消自動更新」→ 取消倒數
- ✅ 開啟「更新設定」→ 更改更新策略

## 🔧 技術改進

### 1. 更新策略系統

**新增 update-config.js：**
- 管理更新策略
- LocalStorage 儲存設定
- 自動/通知/手動三種模式

### 2. 倒數計時機制

```javascript
let countdown = 10;
setInterval(() => {
    countdown--;
    if (countdown <= 0) {
        window.location.reload(); // 自動更新
    }
}, 1000);
```

### 3. 設定持久化

```javascript
localStorage.setItem('update_strategy', 'auto');
```

使用者的設定會永久保存。

## 🐛 疑難排解

### 問題：自動更新沒有觸發

**檢查：**
1. 版本號是否已更新？
2. 更新策略是否設為「自動更新」？
3. 是否等待足夠時間（30 秒）？

**解決：**
- 開啟「更新設定」，確認策略為「自動更新」
- 確認版本號已正確更新
- 清除快取並重新載入

### 問題：想關閉自動更新

**解決：**
1. 點擊 Header 的 **🔄** 按鈕
2. 選擇「顯示通知」或「手動更新」
3. 點擊「儲存設定」

**Q: 自動更新會立即執行嗎？**  
A: 不會，會倒數 10 秒，使用者可隨時取消

**Q: 如果不想自動更新？**  
A: 開啟「更新設定」，選擇「顯示通知」或「手動更新」

A: 不會，所有資料都在 Firebase，只是重新載入頁面

**Q: 可以改倒數時間嗎？**  
A: 可以，修改 update-config.js 中的 `autoUpdateCountdown = 10`
